# [발표 자료 초안] SYNCRIG: 통합 게임 데이터 및 하드웨어 최적화 플랫폼

---

## Slide 1: 타이틀 (Title)
*   **슬라이드 제목:** SYNCRIG: 통합 게이밍 대시보드 및 하드웨어 매칭 시스템
*   **서브 타이틀:** 아이디어 기획부터 클라우드 배포까지의 풀스택 소프트웨어 라이프사이클(SDLC) 여정
*   **발표자:** 이건우 (2021663046)
*   **발표 키워드:** 소프트웨어 공학, 3-Tier 아키텍처, 유사도 추천 엔진, 풀스택 배포
*   **발표 멘트:** "안녕하십니까. 컴퓨터공학과 이건우입니다. 한 학기 동안 진행한 '통합 게임 데이터 및 하드웨어 최적화 플랫폼 - SYNCRIG'의 최종 발표를 시작하겠습니다. 본 발표에서는 단순히 최종 코드만이 아닌, 아이디어 구상부터 실서비스 배포에 이르는 12주간의 소프트웨어 수명 주기 전체를 담아냈습니다."

## Slide 2: [1단계] 추진 배경 및 아이디어 구상 (2~3주차)
*   **핵심 메시지:** 게이밍 경험 파편화와 사양 옵션 타협 스트레스 극복
*   **주요 내용:**
    1.  스팀, 라이엇 등 다중 런처 사용으로 인해 누적 플레이타임 및 도전과제 등 개인 통계 데이터 분산.
    2.  게임 사양 급증으로 인해 내 PC에 적합한 그래픽 옵션을 찾기 위한 탐색 비용(커뮤니티 검색, 벤치마크 영상 시청 등) 과다 발생.
*   **발표 멘트:** "2~3주차에는 게이머들이 일상에서 겪는 두 가지 핵심 페인 포인트를 정의했습니다. 첫째는 여러 플랫폼에 흩어진 전적과 게임 시간의 파편화이고, 둘째는 사양 타협 시 옵션 최적화 탐색 시간이 너무 길다는 점입니다. 이 두 문제를 하나의 플랫폼에서 유기적으로 연결하자는 아이디어에서 출발했습니다."

---

## Slide 3: [2단계] 요구사항 분석 및 와이어프레임 설계 (4~5주차)
*   **핵심 메시지:** SRS(요구사항 정의서) 작성 및 HTML/CSS 프로토타이핑
*   **주요 내용:**
    *   기능 요구사항 정의서(FSD) 수립: 회원가입/로그인, 스팀/라이엇 계정 연동, 하드웨어 프로필 CRUD, 최적 옵션 추천 모달.
    *   Figma 및 HTML 와이어프레임을 이용해 유저의 사용 흐름(User Flow)에 최적화된 초기 화면 구조 기획.
*   **발표 멘트:** "4~5주차에는 기획한 아이디어를 구체적인 기능 정의서와 와이어프레임으로 변환했습니다. 유저가 회원가입 후 로그인하여 하드웨어 스펙을 입력하고, 맞춤형 추천 세팅을 적용하는 일련의 시나리오를 설계하고 기본 정적 프로토타입을 마크업하였습니다."

---

## Slide 4: [3단계] 시스템 디자인 및 데이터베이스 설계 (6주차)
*   **핵심 메시지:** 3-Tier 아키텍처 및 3정규화 PostgreSQL ERD 명세
*   **주요 내용:**
    *   **Architecture:** Stateless Express REST API 서버와 React SPA 클라이언트를 분리하여 느슨한 결합(Loose Coupling) 구현.
    *   **ERD:** Users(사용자), Hardware_Profiles(하드웨어 사양), Optimization_Profiles(추천 프리셋) 간의 관계형 DB 정규화 설계.
*   **발표 멘트:** "6주차에는 3-Tier 아키텍처를 기반으로 시스템 모델을 설계했습니다. DB는 PostgreSQL을 타겟으로 하여 유저당 다중 하드웨어 설정을 가질 수 있게 1대N 관계를 3정규화 릴레이션으로 명세하였으며, 성능 매칭을 위한 API 엔드포인트 규격을 확정했습니다."

---

## Slide 5: [4단계] 프론트엔드 UI 컴포넌트화 및 디자인 고도화 (7주차, 10주차 개선)
*   **핵심 메시지:** 프리미엄 바이올렛(네온 블루/퍼플) 사이버펑크 톤앤매너 테마 적용
*   **주요 내용:**
    *   **UI/UX 개편:** 기존 평범한 단색 레이아웃을 극복하고, 네온 블루(`#3b82f6`)와 네온 퍼플(`#8b5cf6`)을 포인트 컬러로 삼은 고시인성 다크 대시보드 레이아웃 확보.
    *   **컴포넌트 설계:** Multi-step 하드웨어 추가 폼, FPS 예상 그래프, 원클릭 클립보드 복사 모달 등 핵심 요소를 재사용 가능한 리액트 구조로 개발.
*   **발표 멘트:** "7주차와 10주차에는 프론트엔드 구현 및 UI 고도화를 단행했습니다. 게이밍 서비스의 아이덴티티를 살리기 위해 네온 블루와 퍼플이 어우러진 다크 테마를 제작하고, 사양 등록 폼을 3단계 마법사 형태로 컴포넌트화하여 사용자 편의성을 대폭 올렸습니다."

---

## Slide 6: [5단계] 백엔드 API 서버 구축 및 하이브리드 DB 레이어 (8~10주차)
*   **핵심 메시지:** Express 백엔드 서버 구축 및 DB 인메모리 폴백 기능 개발
*   **주요 내용:**
    *   **API 서버:** Express.js 기반의 RESTful 라우팅 설계. PBKDF2 비밀번호 암호화 및 JWT 세션 토큰 연동.
    *   **하이브리드 커넥터:** PostgreSQL 실연동 지원 및 클라우드 외부 DB 오프라인 시에도 인프라에 구애받지 않도록 인메모리 모의(Mock) DB 자동 전환 폴백 로직 구현.
*   **발표 멘트:** "8주차와 10주차에 걸쳐 백엔드 API를 완비했습니다. 로컬 환경이나 DB가 유효하지 않은 프리티어 클라우드 환경에서도 작동이 멈추지 않도록 인메모리 대체 데이터베이스 레이어를 자체 개발해 강건한 예외 처리를 수행하도록 구현했습니다."

---

## Slide 7: [6단계] 핵심 알고리즘 - 유사도 매칭 엔진 구현 (9주차)
*   **핵심 메시지:** 성능 요소별 가중치 연산 모델 수립 (GPU 50%, CPU 30%, RAM 10%, 해상도 10%)
*   **주요 내용:**
    *   텍스트 형태의 CPU/GPU 칩셋 모델명을 파싱하여 세대 및 등급 수치 점수 변환.
    *   유저의 등록 사양과 전체 데이터셋 간의 유클리드 거리 및 가중합 연산을 통한 유사도 백분율 추출.
    *   추천 가중치 임계값(0.8) 필터링 및 추천 수(`likes`) 기반의 랭킹 노출.
*   **발표 멘트:** "9주차에는 서비스의 정수인 하드웨어 유사도 매칭 엔진을 개발했습니다. 그래픽 성능에 지대한 영향을 주는 GPU에 50%, CPU에 30%, RAM과 해상도에 각 10%의 가중치를 두어 연산하는 수학적 모델을 구축해 높은 유사도를 지닌 셋업을 우선적으로 매칭합니다."

## Slide 8: [7단계] 시스템 품질 검증 및 유스케이스 QA (11주차)
*   **핵심 메시지:** 5대 시나리오 QA 패스 및 단위 테스트 단언(Assert)
*   **주요 내용:**
    *   회원가입, 로그인/인증, 프로필 CRUD, 매칭 정확성, 계정 연동 시나리오별 인수 테스트 완료.
    *   매칭 엔진 유사도 계산 단위 테스트 코드(`testCases.js`) 실행을 통해 정량적 무결성 단언 통과.
*   **발표 멘트:** "11주차에는 시스템의 신뢰도 확보를 위해 테스팅을 진행했습니다. 회원가입부터 매칭 조회까지의 대표 유스케이스 5개를 완수하고, 매칭 엔진의 계산 정확도를 검증하는 Assert 기반 단위 테스트를 작성해 100% 기능 성공을 단언하였습니다."

## Slide 9: [8단계] Vercel 서버리스 풀스택 클라우드 배포 (12주차)
*   **핵심 메시지:** Vercel Serverless Function 연동 및 Docker 패키징 이중화
*   **주요 내용:**
    *   **Vercel Full-Stack:** 단일 Git Repo 구조에서 `vercel.json` 설정을 통해 프론트엔드 정적 페이지와 Express API 백엔드(`/api`)를 통합 배포 처리.
    *   **Docker Container:** 프로덕션 환경의 격리 운용을 지원하는 멀티 스테이지 빌드 Dockerfile 작성 완료.
*   **발표 멘트:** "마지막 12주차에는 클라우드 배포 스펙을 갖추었습니다. Vercel Serverless Function을 사용해 백엔드를 가상 서버 환경으로 이식하여 단일 배포 링크로 기동을 성공시켰으며, 클라우드 환경 확장을 위한 Docker 컨테이너 사양도 이중화하여 완료했습니다."
