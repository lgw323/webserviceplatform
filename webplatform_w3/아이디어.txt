# 웹 서비스 기획서: 통합 게임 데이터 및 하드웨어 최적화 플랫폼

## 1. 프로젝트 개요 (Project Overview)

### 1.1. 추진 배경 (Background)

현재 PC 게임 시장은 스팀(Steam), 에픽게임즈(Epic Games), 라이엇 게임즈(Riot Games) 등 다수의 유통 플랫폼(Launcher)으로 파편화되어 있습니다. 이로 인해 사용자는 자신의 총 누적 플레이 시간, 업적(Achievement) 등의 데이터를 통합적으로 관리하기 어렵습니다. 또한, PC 플랫폼의 특성상 사용자마다 하드웨어 제원(CPU, GPU, 모니터 해상도 등)이 상이하여, 특정 게임의 원활한 구동을 위한 '최적화된 그래픽 설정값'을 도출하는 데 높은 탐색 비용이 발생합니다.

### 1.2. 프로젝트 목표 (Objectives)

본 프로젝트의 목표는 분산된 게임 데이터를 하나의 대시보드에 통합 시각화하고, 사용자 개인의 하드웨어 스펙에 기반한 게임별 최적 그래픽 설정(Profile)을 공유 및 매칭하는 '다면 웹 서비스 플랫폼(Multi-sided Web Service Platform)'을 구축하는 것입니다.

### 1.3. 서비스 정의 (Service Definition)

- **데이터 중개 플랫폼:** 외부 게임 유통사의 Open API를 통해 원시 데이터를 수집, 가공하여 시각적 통계 정보로 변환 및 제공합니다.
- **커뮤니티 기반 최적화 허브:** 하드웨어 벤치마크 및 최적화 설정값을 산출하는 '제공자(Power User)'와 이를 자신의 PC에 적용하고자 하는 '수요자(General User)'를 데이터베이스 매칭을 통해 연결합니다.

## 2. 시장 및 현황 분석 (Market & As-Is Analysis)

### 2.1. 시장 동향 및 문제점

- **데이터 파편화:** 복수의 런처 사용이 기본값이 됨에 따라, 크로스 플랫폼 데이터 집계에 대한 수요가 증가하고 있습니다.
- **하드웨어 파편화 및 성능 편차:** 최신 게임의 요구 사양이 급증함에 따라 시스템 병목(Bottleneck) 현상이 빈번하게 발생합니다. 일반적인 권장 사양 표기만으로는 QHD/4K 해상도 및 고주사율 모니터 환경에서의 실제 프레임(FPS) 방어 여부를 판단하기 불가합니다.

### 2.2. 차별화 전략 (Positioning)

기존의 단순 전적 검색 사이트(예: OP.GG)나 하드웨어 커뮤니티(예: 쿨엔조이, 퀘이사존)의 기능을 단일 아키텍처로 통합합니다. 사용자의 하드웨어 프로필(Profile)과 보유 게임 데이터를 교차 분석하여, '보유 중인 게임에 대한 내 PC의 최적 세팅값'을 즉각적으로 도출하는 알고리즘을 제공합니다.

## 3. 서비스 구축 계획 (Service Development Plan)

### 3.1. 시스템 아키텍처 (System Architecture)

본 플랫폼은 데이터 수집의 정확성과 매칭 알고리즘의 효율성을 위해 다음과 같은 3계층 아키텍처(3-Tier Architecture)로 설계됩니다.

- **프론트엔드 (Client-side):** SPA(Single Page Application) 기반으로 구성하여 사용자 경험(UX)의 연속성을 보장하며, 수집된 데이터를 동적 차트로 렌더링합니다.
- **백엔드 (Server-side):** 외부 게임사 API와의 비동기 통신(Asynchronous Request)을 통해 데이터를 수집 및 캐싱(Caching)합니다. RESTful API를 구축하여 클라이언트의 요청을 처리합니다.
- **데이터베이스 (Data Tier):** RDBMS를 활용하여 사용자 정보, 하드웨어 제원, 세팅 프로파일 간의 복잡한 관계 모델을 정규화(Normalization)하여 저장합니다.

### 3.2. 기술 스택 (Tech Stack)

- **Frontend:** React.js (컴포넌트 기반 UI), Tailwind CSS (반응형 스타일링), Recharts (데이터 시각화)
- **Backend:** Node.js (Express.js 프레임워크 기반), Axios (외부 API HTTP 통신)
- **Database:** MySQL 또는 PostgreSQL (관계형 데이터베이스)
- **External API:** Steam Web API, Riot Games API

## 4. 요구사항 및 기능 명세 (Requirements & Functional Specification)

### 4.1. 정보 구조도 (IA)

1. 통합 대시보드 (보유 게임 목록, 총 플레이 타임 시각화)
2. 하드웨어 프로필 관리 (내 PC 사양 등록)
3. 세팅 프로파일 공유 허브 (게임별/하드웨어별 검색 및 필터링)

### 4.2. 핵심 기능 명세

- **OAuth 기반 외부 계정 연동:** 스팀 등 외부 플랫폼의 계정을 연동하여 라이브러리 목록 및 플레이 데이터를 서버로 이관합니다.
- **하드웨어 프로필 등록 시스템:** 사용자가 자신의 시스템 제원을 등록합니다.
    - *(적용 예시: CPU - `AMD Ryzen 9 7950X3D`, GPU - `ASUS TUF Gaming GeForce RTX 4080 SUPER`, Monitor - `LG 27GP850 (QHD 165Hz)`)*
- **알고리즘 기반 필터링 매칭:** 사용자가 특정 게임(예: 사이버펑크 2077)을 선택하면, 데이터베이스에 저장된 프로파일 중 사용자의 등록된 하드웨어 환경(예: 7950X3D + 4080 SUPER + QHD)과 90% 일치하는 환경에서 작성된 '최적 그래픽 설정값' 및 '평균 예상 프레임(FPS)' 데이터를 우선순위로 정렬하여 출력합니다.

## 6. 수익 창출 모델 (Business Model)

### 6.1. B2B 정밀 타겟팅 광고 및 제휴 마케팅 (Targeted Ads & Affiliate)

- **논리:** 플랫폼에 누적된 사용자의 '하드웨어 제원 데이터'를 마케팅 자산으로 활용합니다.
- **구현 방안:** 사용자의 현재 GPU(예: `RTX 4080 SUPER`) 및 시스템 구성을 분석하여, 성능 병목을 유발하는 노후 부품에 대한 업그레이드 광고를 노출하거나, 해당 사양에 적합한 고해상도/고주사율 모니터 광고를 송출합니다. 하드웨어 세팅 프로파일 내 부품 구매 링크(오픈마켓) 연동을 통한 판매 수수료(Commission)를 창출합니다.

### 6.2. B2C 프리미엄 구독 (Premium Subscription)

- **논리:** 플랫폼의 고도화된 편의성을 요구하는 파워 유저를 대상으로 부분 유료화(Freemium) 전략을 채택합니다.
- **구현 방안:** 기본 대시보드는 무료로 제공하되, '인게임 그래픽 설정 파일(Config)의 클라우드 자동 백업 및 복원', '데이터 동기화 API 호출 우선순위 할당', '상위 1% 플레이어와의 하드웨어 성능 및 전적 교차 분석 통계' 등의 심화 기능을 월 구독 형태로 제공합니다.

### 6.3. B2B 익명화 데이터 판매 (Data Monetization)

- **논리:** 특정 게임에 대한 대규모 유저군의 실제 프레임(FPS) 지표 및 하드웨어 점유율 데이터를 정보 자산화합니다.
- **구현 방안:** 게임 개발사(QA/최적화 부서) 또는 하드웨어 제조사 측에 시장 리서치용 B2B API 또는 정기 리포트 형태로 통계 데이터를 판매합니다.

## 7. 기대 효과 및 향후 계획 (Expected ROI & Future Plan)

### 7.1. 기대 효과

- **사용자 측면:** 파편화된 게임 기록의 일원화 및 하드웨어 성능 타협에 소요되는 시간적 비용을 획기적으로 단축합니다.
- **플랫폼 측면:** 데이터 중개 및 매칭이라는 웹 서비스의 본질적 아키텍처를 성공적으로 구현하여 시스템 설계 역량을 입증합니다.

### 7.2. 향후 확장 계획

초기 모델은 PC 게임에 국한되나, 향후 아키텍처 확장을 통해 콘솔 플랫폼(PlayStation Network, Xbox Live)의 데이터 연동 및 콘솔 기기별 최적화 모니터 세팅 정보 공유 등으로 도메인을 확장할 수 있습니다.
