Module 1: Design System & UI Framework너는 전문적인 프론트엔드 개발자이자 UI/UX 디자이너다. 아래의 디자인 시스템 명세서를 바탕으로 React(Vite), Tailwind CSS, 그리고 shadcn/ui 컴포넌트를 활용하여 웹 애플리케이션의 시각적 뼈대를 구축하라.1. 전반적인 UI/UX 디자인 가이드 (Design System)Theme: 게이머를 타겟으로 한 모던 다크 테마 (Modern Dark Mode). 배경은 짙은 회색(#121212) 또는 네이비 톤을 사용하고, 포인트 컬러로 네온 블루(#00f0ff) 또는 일렉트릭 퍼플(#b026ff)을 사용하여 사이버네틱한 느낌을 부여할 것.Layout: 로그인/랜딩 페이지를 제외한 모든 내부 페이지는 좌측 글로벌 내비게이션 사이드바(Sidebar)와 우측 메인 콘텐츠로 구성된 Dashboard 레이아웃을 적용.Components: * 데이터 시각화: Recharts 라이브러리 사용.아이콘: Lucide React 사용.기본 UI 요소: shadcn/ui (카드, 버튼, 폼, 데이터 테이블, 모달) 적극 활용.

Module 3: Interaction & Logic Requirements구축된 UI 화면(Module 2)을 기반으로, 클라이언트 사이드의 상태 관리 및 라우팅 등 논리적 상호작용(Interaction) 요소를 구현하라.기능적 요구사항 (Interaction & Logic)Routing: 좌측 사이드바를 통한 페이지 간 클라이언트 사이드 라우팅(Client-side Routing, React Router) 구현. 화면 전환 시 지연 없는 SPA(Single Page Application) 경험 제공.State Management: 랜딩 페이지에서 로그인을 모사하는 버튼 클릭 시, 인증 상태(Auth State)를 업데이트하고 대시보드로 이동하며 사이드바가 나타나는 상태 전환 로직 처리.Responsive Design: * 데스크톱 환경에서는 전체 대시보드 구조 유지.모바일 환경(Breakpoint 적용)에서는 사이드바가 햄버거 메뉴(Hamburger Menu)로 숨겨지며 토글 시 오버레이(Overlay) 형태로 나타나는 반응형 레이아웃 필수 적용.
